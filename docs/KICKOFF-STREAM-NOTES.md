## Judging Criteria (Prioritized from Stream)

The stream clarified the **relative importance** of criteria:

### 1. Application of Technology (MOST IMPORTANT)
- Use Band as the **coordination layer** — this is THE requirement
- Judges look for:
  - Clear task handoffs between agents
  - Shared context passing
  - Role specialization
  - Task state management through Band
- **"When a judge looks at your project, they should see RIGHT AWAY which agent did what and where the handoffs happened"**

### 2. Presentation
- How clearly the team explains and demonstrates the workflow
- **Pro tip from stream**: Check previous hackathon winners on lablab.ai for presentation style reference

### 3. Business Value
- How well the project solves a real enterprise problem

### 4. Originality
- How creatively agents discover each other
- How they divide work between themselves
- Collaboration across frameworks

## Critical Advice from Speakers

### From Offer (Band - Head of DevRel)
> "The key thing we want to see is agents actually working together meaningfully. They should be communicating, sharing context, handing off work, and making decisions."

- Pick a **real workflow**
- **Make the collaboration visible**
- Show why **multiple agents are better than one**
- Human-in-the-loop is valid: human joins Band chat room to approve steps, resolve ambiguity, or guide the system

### From Valerie (AI/ML API - Product Lead)
> "The track pick matters less than this one thing: Make the collaboration visible."

Key insights:
- **Different agents want different brains** — planner needs reasoning model, coder needs code model, summarizer needs cheap/fast model
- AI/ML API = one key, OpenAI-compatible, switch models by changing one string
- **500+ models available through single API**
- For the AI/ML API prize: agents must **actually run the API doing real work** in the system, not one call at the end

## Strategic Insights for Our Build

### What Wins (synthesized from all speakers):
1. **Visible collaboration** — judges must immediately see which agent does what
2. **Clear handoffs** — show the work flowing between agents
3. **Real enterprise problem** — not a toy demo
4. **Cross-framework usage** — shows you understand Band's value prop
5. **Multiple model types** — use reasoning models for planning, code models for coding, fast models for summarization (AI/ML API makes this easy)
6. **Human-in-the-loop** — valid and impressive if done well

### What Doesn't Win:
- Agents bolted onto one script
- Band used as thin wrapper / notification system / output channel
- Single call to partner APIs at the end
- Private GitHub repos (judges can't review)
- Generic agent names (degrades LLM routing)

### Maximizing Prize Potential:
- Can win BOTH main prize AND partner prizes simultaneously
- Use AI/ML API meaningfully throughout → eligible for $1,000 cash + $1,000 credits bonus
- Use Featherless for specialized open-source models → eligible for their prize track
- Use Band as core coordination → eligible for main $3,500/$2,500/$1,500
